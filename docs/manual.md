# WebFT8 PWA 運用マニュアル

**[English version](manual.en.md)** | **[アプリを開く](https://jl1nie.github.io/webft8/)**

WebFT8 はブラウザだけで FT8 の QSO が完結する PWA です。デコード、送信、CAT 制御、ログ管理まで搭載。インストール不要で、Chrome / Edge / Safari (WebKit) で動作します。

---

## 画面構成

```
+--------------------------------------+
| WebFT8 [Scout][Snipe] 14.074  12s * | <- ヘッダー（モード/バンド/残秒/設定）
|--------------------------------------|
| ::::::::: Waterfall ::::::::::::::::: | <- ウォーターフォール（タップで DF 設定）
| ::::::::|::::::::::::::::::::::::::::: |    赤い縦線 = 現在の DF
|--------------------------------------|
| oooo CALLING 3Y0Z  12 decoded(213ms)| <- Scout ステータスバー
|--------------------------------------|
| --- 12:30 -------------------------- | <- ピリオド区切り（UTC インライン）
|  1234 +0.3  -12  CQ 3Y0Z JD34       | <- DF  DT  SNR  メッセージ
|   800 +0.1   -8  JA1ABC 3Y0Z PM95   |
|--------------------------------------|
| [CQ]  [Halt]  Auto                  | <- TX アクション（動的）
+--------------------------------------+
```

## 初回セットアップ

1. [アプリ](https://jl1nie.github.io/webft8/) を開く
2. 設定パネル（右上の歯車アイコン）が自動で開く（コールサイン未設定時のみ）
3. 以下を入力:
   - **My Callsign** -- 自局コールサイン（例: `JA1ABC`）。自動で大文字変換
   - **My Grid** -- グリッドロケータ（例: `PM95`）。自動で大文字変換
   - **Audio Input** -- USB オーディオインタフェース（受信側）を選択
   - **Audio Output** -- TX 音声の出力先デバイスを選択（リグへの送信側）
   - **RX / TX ゲイン** -- 入出力レベルをスライダーで調整（後述）
4. ヘッダーの **Band** セレクタで運用バンドを選択（例: `14.074 (20m)`）
5. **ロゴ（WebFT8）をタップ** → ライブデコード開始（ロゴが青く点灯）

> オーディオデバイスの選択は localStorage に保存され、次回起動時に自動復元されます。2 回目以降はロゴをタップするだけで開始できます。

## 2 つのモード

### Scout モード -- ゆるい CQ 運用

メッセージアプリのようなチャット UI で、気軽に QSO できます。移動運用やスマホでの使用に最適。

**基本操作:**

| 操作 | 動作 |
|------|------|
| **CQ ボタン** | 次のピリオドで CQ を発信（TX キューに入る） |
| **ウォーターフォールをタップ** | TX 周波数（DF）を設定。赤い縦線で表示 |
| **受信メッセージをタップ** | その局の送信元（call2）を呼ぶ（次ピリオドで TX） |
| **Auto チェック** | ON: 自動応答。OFF: TX メッセージを手動選択 |
| **Halt** | 送信を即座に停止 |

**Scout ステータスバー（WF とリストの間）:**

```
oooo CALLING  3Y0Z   |   12 decoded (213ms)
```

- 進捗ドット（4 段階: 灰=未、青=現在、緑=完了）
- QSO 状態 + DX コールサイン
- デコード数 + デコード時間
- IDLE 時: デコード情報のみ表示

**メッセージ表示:**

```
--- 12:30 --------------------------
 1234  +0.3  -12  CQ 3Y0Z JD34
  800  +0.1   -8  JA1ABC 3Y0Z PM95
--- 12:45 --------------------------
 1500  +0.2   -5  CQ VK2RG QF56
```

- **DF / DT / SNR / Message** の統一カラムフォーマット
- ピリオド境界を UTC タイムスタンプ付きの線で区切り（デコード 0 局時はスキップ）
- 青枠 = RX、グレー枠 = TX
- **黄色太字** = 自局コールサイン、**緑太字** = QSO 相手

**QSO の流れ（自動モード）:**

1. ウォーターフォールをタップして DF を決める
2. CQ ボタンで CQ を次ピリオドにキュー、または受信メッセージをタップして呼ぶ
3. 相手から応答 → 自動でレポート交換
4. RR73 / 73 → QSO 完了（ログに自動保存）

**Scout の AP デコード:**

QSO 中（相手局が確定している状態）では、A Priori (AP) デコードが自動的に有効になり、相手局の微弱な信号をより確実に受信します。AP ターゲットは QSO の相手局から自動設定されます（手動入力不要）。

---

### Snipe モード -- DX ハンティング

ターゲット局を追い込む専用モード。ウォーターフォールが大きく表示されます。

Snipe モードの核心は **DF（TX 周波数）と BPF（RX 窓中心）の分離** です。Watch フェーズで送信周波数を決め、Call フェーズでは受信窓だけを動かしてターゲットを追跡できます。これにより、DX 局の周波数とは異なる周波数で呼ぶ「オフ周波数コール」が可能です。

#### Watch フェーズ（全帯域受信）

ターゲットを探し、**送信周波数（DF）** を決めるフェーズ。受信は全帯域（100--3000 Hz）。

| 操作 | 動作 |
|------|------|
| **ウォーターフォールをタップ** | **DF（TX 周波数）** を設定（赤い縦線で表示） |
| **受信メッセージをタップ** | call1 をターゲットに設定（AP デコードが自動有効化）。TX アクションに call2（送信元）を呼ぶボタンも表示 |

**Phase ヒント:** ステータスに `full-band  DF 1200 Hz` と表示

**Watch 画面の表示:**

- **上部**: ターゲット局の最新メッセージ（コール / 周波数 / SNR）
- **Callers**: ターゲットを呼んでいる他局の一覧
- **メッセージ一覧**: DF / DT / SNR / Message の統一フォーマット（ピリオド区切り付き）
- **QSO 進捗ドット**: ●○○○ → ●●○○ → ●●●○ → ●●●●

#### Call フェーズ（ナローフィルタ受信）

Watch で DF を決めたら **Call** ボタンで切り替え。CAT 接続時はリグの DSP ナローフィルタ (FIL3) が自動で有効になり、VFO 周波数も BPF 窓中心に連動してシフトします。

| 操作 | 動作 |
|------|------|
| **ウォーターフォールをタップ** | **BPF（RX 窓中心）** を設定。DF（TX 周波数）は Watch で決めた値を維持。CAT 接続時はリグの VFO も連動 |

**Phase ヒント:** ステータスに `BPF 1050 Hz  DF 1200 Hz` と表示

- 500 Hz BPF 窓が水色の帯で表示される
- 自分とターゲットの交信だけが表示される（ノイズ削減）
- Watch でターゲットをタップ済みなら、QSO ステートに従い Auto で自動応答が継続
- Call に切り替えただけでは TX は開始しない（Watch で先にターゲットをタップして QSO を開始する）
- QSO 失敗（リトライ上限）→ 自動で Watch に戻り、リグのフィルタもワイドに復帰
- 手動で Watch に戻って DF を変更することも可能

**VFO 連動（CAT 接続時）:**

BPF 窓を移動すると、リグの VFO 周波数が自動でシフトし、物理フィルタがターゲット信号をカバーするように調整されます。Watch に戻ると VFO は元のバンド周波数に復帰します。

**Watch / Call の切り替え:**

Snipe ビュー上部の `[Watch] [Call]` タブで切り替え。リストはクリアされず維持されます。Watch に戻ると CAT フィルタはワイド (FIL2) に自動復帰します。

## ウォーターフォール

100-3000 Hz のリアルタイムスペクトログラム。Scout: レスポンシブ高さ（min(25vh, 220px)）、Snipe: 280px。

| 要素 | 説明 |
|------|------|
| **上部ラベル** | 周波数軸（200, 500, 1000, 1500, 2000, 2500 Hz） |
| **赤い点線（縦）** | DF（TX 周波数）-- 全モードでタップ時に表示 |
| **水色の帯** | Snipe Call フェーズの 500 Hz BPF 窓 |
| **赤い点線（横）** | ピリオド境界（15 秒区切り） |
| **黄色テキスト** | デコードされたメッセージ |
| **タップ / クリック** | DF 設定 + ステータスバーに周波数表示 |
| **WAV ドラッグ＆ドロップ** | オフライン解析（ライブ中でも自動停止して処理） |

### FT8 信号の見え方

FT8 信号は ~50 Hz 幅の縦帯として表示され、**1 シンボル (160 ms) ごとに 8 つのトーンの間を跳び移る**滑らかな波打ち模様になります。これは 8-GFSK 変調の per-symbol トーン跳びがそのまま視覚化された **正常動作**です。

WSJT-X はもっと長い FFT 積分窓 (~2.7 秒 = ~17 シンボル) を使うので、シンボル毎のトーン跳びが平均化されて均一な縦帯に見えます。WebFT8 の rendering は per-symbol で「正直」、WSJT-X は per-frame で「整理済み」、と思ってください。表示の違いだけでデコード品質には影響しません。

## TX アクション（ボトムバー）

全ての TX は**次のピリオド境界に同期**してキューされます（即時送信はしない）。

| QSO 状態 | Auto ON | Auto OFF |
|----------|---------|----------|
| **IDLE** | `[CQ]` ボタン | `[CQ]` ボタン |
| **QSO 中** | 次の TX メッセージ 1 つ | 全 TX 選択肢をボタン表示 |

- **Snipe**: ターゲットと異なる局（call2）を呼ぶ `[Call xxx]` ボタンも表示
- **Halt / Reset（プログレッシブ）**: 1 回目タップ = TX 即停止（ボタンが「Reset」に変化）、2 回目タップ = QSO をリセット（incomplete としてログ保存）
- ステータスバーに `TX queued: ...` と表示

---

## QSO ステートマシン

4 段階の状態遷移で QSO を管理:

```
IDLE -> CALLING -> REPORT -> FINAL -> IDLE (完了)
```

| 状態 | 意味 | 送信内容 |
|------|------|----------|
| **IDLE** | 待機中 | -- |
| **CALLING** | 呼び出し中 | `DX MYCALL GRID` or `CQ MYCALL GRID` |
| **REPORT** | レポート交換中 | `DX MYCALL R+00` |
| **FINAL** | 確認待ち | `DX MYCALL RR73` or `73` |

- **Auto ON**: 状態遷移は全自動。応答がなければリトライ（最大 15 回）。
- **Auto OFF**: TX メッセージ選択ボタンが表示され、手動で選択。
- リトライ上限に達すると QSO は incomplete としてログに記録。

---

## 適応バジェット（Scout モード）

Scout モードでは、デコード時間が FT8 バジェット（2.4 秒）を超えると、重い機能から順に自動で一時停止します:

| 優先度 | 機能 | 負荷 |
|--------|------|------|
| 1（先に OFF） | Multi-pass subtract | 3 パス SIC（最も重い） |
| 2（次に OFF） | A Priori (AP) | Sniper + EQ 1 パス |

ステータスバーに `[-sub]` や `[-sub,AP]` と表示されます。

デコード時間がバジェットの 60% 以下に戻ると、逆順で復帰:
1. AP を先に復帰
2. 余裕があれば subtract を復帰

Snipe モードは帯域が狭く高速なため、常に両方有効です。

## コールサインハッシュ解決

FT8 メッセージの一部（Type 4 非標準コール、DXpedition メッセージ等）では、コールサインが 10/12/22 ビットのハッシュとして送信されます。通常は `<...>` と表示されますが、WebFT8 は**デコード済みコールサインのハッシュテーブル**を自動で構築し、以降のメッセージで `<JA1ABC>` のように実コールを表示します。

テーブルは WASM セッション中に蓄積され（最大 1000 エントリ、LRU）、ページリロードでクリアされます。

## オーディオレベル調整

設定パネルの Station セクションに RX / TX のレベル調整があります。

| 項目 | 説明 |
|------|------|
| **RX ゲインスライダー** | 受信入力レベル（0--200%）。GainNode で信号増幅/減衰 |
| **RX レベルメーター** | リアルタイムのピークレベル表示（緑 = 正常、赤 = クリップ）。スライダー操作は即座に反映 |
| **TX ゲインスライダー** | 送信出力レベル（0--200%） |
| **TX レベルメーター** | Test Tone 再生中および送信時のピークレベル表示。スライダー操作は即座に反映 |
| **CLIP 表示** | デジタル波形のピークが 95% を超えると赤い **CLIP** が点灯 |
| **Test Tone ボタン** | 現在の DF 周波数でサイン波を連続再生。タップで開始/停止。リグ側の入力レベル調整に使用 |

### RX レベル調整

過大入力はデコード失敗の主要原因です。RX メーターが常に赤（CLIP）なら、RX ゲインを下げるか、リグのオーディオ出力レベルを調整してください。

### TX レベル調整

**CLIP 表示はアプリ内のデジタル歪みのみを示します。** TX ゲイン 100% 以下ではデジタル歪みは発生しません。

ただし、リグへの実際の入力レベルはアプリの外側で決まります:

- **スマホの OS メディア音量** -- Android の音量スライダー
- **USB オーディオ DAC の出力レベル** -- デバイス固有
- **リグの MOD 入力ゲイン** -- アナログ段の設定

これらはアプリの CLIP 表示には反映されません。Test Tone を流しながら**リグの ALC メーター**を確認し、ALC が振れないレベルに OS 音量と TX ゲインを調整してください。

ゲイン設定は localStorage に保存され、次回起動時に復元されます。

## 設定パネル

歯車アイコンで開閉。アコーディオン形式で 5 セクション（必須度・使用頻度順）。

### Station

| 項目 | 説明 |
|------|------|
| **My Callsign** | 自局コールサイン（入力時に自動大文字変換） |
| **My Grid** | グリッドロケータ（4 桁、自動大文字変換） |

### Rig

| 項目 | 説明 |
|------|------|
| **Audio Input** | 受信用オーディオデバイス（選択は自動保存） |
| **Audio Output** | TX 音声の出力先デバイス（選択は自動保存） |
| **RX / TX ゲイン** | 入出力レベルスライダー + レベルメーター |
| **Test Tone** | ALC 調整用連続キャリア送出（CAT PTT 連動） |
| **Start Audio** | ライブデコードの開始・停止（ロゴタップでも可） |
| **Rig Model** | リグ選択ドロップダウン（rig-profiles.json から動的生成） |
| **Connect Rig** | Web Serial でリグに接続。デスクトップ Chrome / Edge |
| **Connect BLE** | Web Bluetooth で IC-705 に BLE 接続。スマホ対応（後述） |

### Log

| 項目 | 説明 |
|------|------|
| **Export ZIP (ADIF + RX)** | QSO ログと RX ログを ZIP でダウンロード |
| **Clear All Logs** | 全ログ削除（確認ダイアログあり） |

### TX Messages

| 項目 | 説明 |
|------|------|
| **Tx1 CQ suffix** | CQ サフィックス（POTA / SOTA / DX 等、4 文字以内） |
| **Tx5 Free text** | フリーテキスト送信（SOTA リファレンス等、13 文字）。RR73 の代わりに送信 |

### Decode

| 項目 | 説明 |
|------|------|
| **Strictness** | デコード感度 vs 偽陽性のバランス（Strict / Normal / Deep） |
| **Equalizer** | 適応型イコライザ（Off / Adaptive）。BPF エッジ補正 |
| **Retry limit** | QSO リトライ回数上限（デフォルト 15） |
| **Multi-pass subtract** | 逐次干渉除去（3 パス SIC）。デフォルト ON |
| **A Priori (AP)** | AP デコード。デフォルト ON |
| **CQ reply: best SNR** | ON: 最強 SNR の CQ に応答。OFF: 最初にデコードされた CQ に応答 |
| **Waterfall FFT** | ウォーターフォール表示の ON/OFF |
| **Open WAV File** | WAV ファイルを選択してオフライン解析 |

> AP ターゲットは手動入力不要。Scout では QSO 中の相手局から、Snipe ではタップしたターゲット局から自動設定されます。

### パイプラインデコード

Subtract ON 時、デコードは 2 フェーズで実行されます:

1. **Phase 1**（~200ms）: 強信号を高速デコード → 結果を即座にチャットに表示
2. **Phase 2**（~300-800ms）: 3 パス減算で追加信号を検出 → チャットに追記

Phase 1 の FFT キャッシュを Phase 2 で再利用し、計算コストを削減。バジェット超過時は Phase 2 をスキップ（強信号は Phase 1 で取得済み）。

## ログ管理

### 自動保存

- **QSO 完了時**: 自動で localStorage に保存（state = IDLE）
- **QSO 中断時**: Reset ボタン or リトライ上限で incomplete として保存（state = CALLING/REPORT/FINAL）
- **全受信メッセージ**: 全デコード結果を RX ログとして蓄積（最大 10,000 件）

### ZIP エクスポート

設定パネルの **Export ZIP (ADIF + RX)** ボタンで、以下の 3 ファイルを ZIP ダウンロード:

| ファイル | 内容 |
|----------|------|
| `qso_complete_YYYYMMDD.adi` | 完了 QSO のみ（LoTW / HAMLOG 投入用） |
| `qso_all_YYYYMMDD.adi` | 全 QSO（incomplete には `<COMMENT>incomplete:STATE</COMMENT>` 付き） |
| `rx_YYYYMMDD.csv` | 全受信デコードログ（UTC, Freq, SNR, Message） |

### ログのクリア

**Clear All Logs** ボタンで QSO ログと RX ログの両方を削除（確認ダイアログあり）。

## WAV ファイルでのオフライン解析

ウォーターフォールに WAV ファイルをドラッグ＆ドロップするか、設定パネルの **Open WAV File** で選択。

- **要件**: 16 bit / mono の WAV（サンプルレートは任意 — 12 / 44.1 / 48 kHz など。Rust 側で自動リサンプル）
- ライブデコード中でも自動停止して処理
- ウォーターフォール + デコード結果が即座に表示

## CAT 制御

### Web Serial（デスクトップ）

Web Serial API を使い、USB 経由でリグの PTT・フィルタ・周波数・モードを制御します。

**対応リグ:**

| プロトコル | 対応リグ |
|-----------|---------|
| **Yaesu テキスト CAT** | FTDX10, FTDX101MP, FT-891 |
| **Icom CI-V** | IC-7300, IC-705, IC-7851 |

**接続手順:**

1. USB ケーブルでリグと PC を接続
2. 設定パネルの CAT セクションで **Rig Model** を選択
3. **Connect CAT** → ブラウザのシリアルポート選択ダイアログが開く
4. ポートを選択 → 接続完了

### IC-705 BLE 接続（スマホ / ワイヤレス）

IC-705 は BLE (Bluetooth Low Energy) で CI-V CAT コマンドを送受信できます。ケーブルなしでスマホから PTT や周波数を制御できます。ただし音声は BLE では送れないため、USB オーディオケーブルまたは 3.5mm ケーブルが別途必要です。

> **BLE = CAT 制御のみ。** 音声（RX/TX）は BLE を通りません。USB OTG ケーブルまたは ACC/SP 端子で接続してください。

#### 初回セットアップ

BLE 接続は OS のペアリング不要です。以下の手順だけで接続できます。

1. **IC-705 側の操作:** MENU → **SET** → **Bluetooth** → **Bluetooth** = **ON**、**相手からのペアリング要求** をタップして待機
2. **WebFT8 の設定**（歯車アイコン）→ **Rig** セクションでリグを選択し、**Connect BLE** をタップ
3. ブラウザのデバイス選択ダイアログで「ICOM BT(IC-705)」を選択
4. ステータスに「connected」と表示 → 接続完了！

#### 2回目以降の接続

一度ペアリングすれば IC-705 がデバイスを記憶します:

1. IC-705 の Bluetooth が ON であることを確認（通常 ON のまま）
2. WebFT8 でロゴタップ（または設定の **Connect BLE**）
3. ダイアログで IC-705 を選択 → 数秒で接続

#### 必要条件

| 項目 | 詳細 |
|------|------|
| **ブラウザ** | Android Chrome（Web Bluetooth 必須） |
| **iOS** | Safari は Web Bluetooth 非対応。[Bluefy](https://apps.apple.com/app/bluefy-web-ble-browser/id1492822055) ブラウザを使用 |
| **位置情報** | Android は BLE スキャンに位置情報 (GPS) の ON が必要 |
| **音声** | USB オーディオケーブルまたは 3.5mm ケーブルが別途必要 |

#### トラブルシューティング

| 症状 | 対処 |
|------|------|
| IC-705 がダイアログに表示されない | スマホの位置情報 (GPS) を ON にする |
| ペアリング失敗 | 両方のデバイスで古いペアリング情報を削除し、手順 1 からやり直し |
| 接続済みだが PTT が効かない | IC-705 の CI-V アドレス（デフォルト 0xA4）を確認。DATA モードであること |
| 音声が出ない | BLE は CAT のみ。USB オーディオケーブルを別途接続 |

### CAT 自動制御

| 機能 | タイミング |
|------|-----------|
| **PTT ON/OFF** | TX 開始/終了時に自動制御 |
| **モード設定** | バンド変更時に DATA-USB (FIL2) を自動設定 |
| **ナローフィルタ** | Snipe Call フェーズ開始時に FIL3 (500Hz) を自動適用 |
| **ワイドフィルタ** | Snipe Watch フェーズ復帰時に FIL2 (2400Hz) に自動復帰 |
| **VFO 周波数** | バンド変更時に自動設定。Snipe Call 中は BPF 窓位置に連動 |

**Icom リグのフィルタ設定（事前準備）:**

CI-V コマンドは FIL2 / FIL3 の切り替えを行います。フィルタ幅はリグ本体で設定してください:
- **FIL2**: 2400 Hz（ワイド用）
- **FIL3**: 500 Hz（ナロー用）

設定: MENU → Set → Filter → DATA-USB → FIL2 / FIL3 の幅を変更

> Web Serial API は Chrome / Edge でのみ利用可能です。Safari / Firefox は非対応。

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| ロゴが暗いまま | ロゴをタップして Audio Start。設定パネルで Audio Input が選択されているか確認 |
| デコード結果が 0 件 | アンテナ接続、オーディオレベル（CLIP していないか確認）、周波数（14.074 MHz 等）を確認 |
| RX メーターが常に CLIP | RX ゲインを下げるか、リグのオーディオ出力レベルを下げる |
| WAV ドロップでエラー | 16 bit / mono か確認（サンプルレートは任意） |
| CAT 接続できない | Chrome / Edge を使用。ブラウザ再起動後に再試行 |
| `port busy or unresponsive` 表示 | 他の CAT ソフト（WSJT-X 等）が同じ COM ポートを掴んでいる。終了してから再接続 |
| BLE で IC-705 が見つからない | スマホの位置情報が ON か確認。IC-705 の Pairing Reception が ON か確認。OS 設定で先にペアリング |
| CAT disconnected が出る | コマンドの送信間隔が短すぎる可能性。ブラウザ再起動後に再接続 |
| ウォーターフォールが真っ黒 | Audio Input が正しいデバイスか確認。ロゴタップで再起動 |
| QSO が進まない | Auto チェックが ON か確認。受信メッセージをタップして相手局を選択 |
| `[-sub]` `[-sub,AP]` 表示 | デコードが重いため自動調整中。バジェット内に収まれば復帰 |
| `<...>` が解決されない | 同一セッション中にその局がデコードされていない。ページリロードでハッシュテーブルはクリア |

## PWA インストール（オフライン対応）

WebFT8 は PWA としてデバイスにインストールでき、オフラインでも動作します。

**インストール方法:**

| ブラウザ | 手順 |
|---------|------|
| **Chrome (PC)** | アドレスバー右のインストールアイコン、または メニュー → 「アプリをインストール」 |
| **Chrome (Android)** | メニュー → 「ホーム画面に追加」 |
| **Safari (iOS)** | 共有ボタン → 「ホーム画面に追加」 |
| **Edge** | メニュー → 「アプリ」→「このサイトをアプリとしてインストール」 |

インストール後はホーム画面 / アプリ一覧からスタンドアロンで起動できます。初回オンライン時にファイルがキャッシュされ、以降はオフラインでも WAV 解析が可能です（ライブオーディオにはデバイスのマイク許可が必要）。

> アプリの更新: オンライン接続時に自動的に最新版がダウンロードされます。
