# rs-ft8n PWA 運用マニュアル

**[English version](manual.en.md)** | **[アプリを開く](https://jl1nie.github.io/rs-ft8n/)**

rs-ft8n はブラウザだけで FT8 の QSO が完結する PWA です。デコード、送信、CAT 制御、ログ管理まで搭載。インストール不要で、Chrome / Edge / Safari (WebKit) で動作します。

---

## 画面構成

```
+--------------------------------------+
| rs-ft8n [Scout][Snipe]    12s (213ms)| <- ヘッダー（モード、残秒、デコード時間）
|--------------------------------------|
| ::::::::: Waterfall ::::::::::::::::: | <- ウォーターフォール（タップで DF 設定）
| ::::::::|::::::::::::::::::::::::::::: |    赤い縦線 = 現在の DF
|--------------------------------------|
|  1234 +0.3  -12  CQ 3Y0Z JD34       | <- DF  DT  SNR  メッセージ
|   800 +0.1   -8  JA1ABC 3Y0Z PM95   |
|--------------------------------------|
| [CQ]  [Halt] Auto    12 decoded     | <- TX アクション + ステータス
+--------------------------------------+
```

## 初回セットアップ

1. [アプリ](https://jl1nie.github.io/rs-ft8n/) を開く
2. 設定パネル（右上の歯車アイコン）が自動で開く
3. 以下を入力:
   - **My Callsign** — 自局コールサイン（例: `JA1ABC`）
   - **My Grid** — グリッドロケータ（例: `PM95`）
   - **Audio Input** — USB オーディオインタフェース（受信側）を選択
   - **Audio Output** — TX 音声の出力先デバイスを選択（リグへの送信側）
4. **Start Audio** をタップ → ライブデコード開始

> CAT 制御を使う場合は、CAT Protocol を選び **Connect CAT** をタップ。Web Serial API 対応ブラウザが必要です（Chrome / Edge）。

## 2 つのモード

### Scout モード — ゆるい CQ 運用

メッセージアプリのようなチャット UI で、気軽に QSO できます。移動運用やスマホでの使用に最適。

**基本操作:**

| 操作 | 動作 |
|------|------|
| **CQ ボタン** | 次のピリオドで CQ を発信（TX キューに入る） |
| **ウォーターフォールをタップ** | TX 周波数（DF）を設定。赤い縦線で表示 |
| **受信メッセージをタップ** | その局の送信元（call2）を呼ぶ（自動で QSO 開始） |
| **Auto チェック** | ON: 自動応答。OFF: TX メッセージを手動選択 |
| **Halt** | 送信を即座に停止 |

**メッセージ表示（統一フォーマット）:**

```
 DF(Hz)  DT(s)  SNR  Message
  1234   +0.3   -12  CQ 3Y0Z JD34
   800   +0.1    -8  JA1ABC 3Y0Z PM95
```

- 青枠の行 = 受信メッセージ（RX）
- グレー枠の行 = 送信メッセージ（TX）
- **黄色太字** = 自局コールサイン
- **緑太字** = QSO 相手のコールサイン

**QSO の流れ（自動モード）:**

1. ウォーターフォールをタップして DF を決める
2. CQ ボタ���で CQ を次ピリオドにキュー、または受信メッセージをタップして呼ぶ
3. 相手から応答 → 自動でレポート交換
4. RR73 / 73 → QSO 完了（ログに自動保存）

**Scout の AP デコード:**

QSO 中（相手局が確定している状態）では、A Priori (AP) デコードが自動的に有効になり、相手局の微弱な信号をより確実に受信します。デコード時間が FT8 バジェット（2.4 秒）を超える場合は自動的に AP と subtract が一時停止されます（[適応バジェット](#適応バジェットscout-モード)参照）。

### Snipe モード — DX ハンティング

ターゲット局を追い込む専用モード。ウォーターフォールが大きく表示されます。

#### Watch フェーズ（全帯域受信）

ターゲットを探し、呼ぶ周波数を決めるフェーズ。

| 操作 | 動作 |
|------|------|
| **DX Call (AP)** を設定パネルで入力 | ターゲット局を指定（AP デコードが有効化） |
| **ウォーターフォールをタップ** | DF を設定（赤い縦線で表示）。Call フェーズではこの周波数が 500 Hz 窓の中心になる |
| **受信メッセージをタップ** | call1 をターゲットに設定。TX アクションに call2（送信元）を呼ぶボタンも表示 |

**Watch 画面の表示:**

- **上部**: ターゲット局の最新メッ���ージ（コール / 周波数 / SNR）
- **Callers**: ターゲットを呼んでいる他局の一覧
- **メッセージ一覧**: DF / DT / SNR / Message の統一フォーマット
- **QSO 進捗ドット**: ●○○○ → ●●○○ → ●●●○ → ●●●●

#### Call フェーズ（呼び続ける）

Watch で DF を決めたら **Call** ボタンで切り替え。

- 500 Hz BPF 窓が水色の帯で表示される
- 自分とターゲットの交信だけが表示される（ノイズ削減）
- 自動で呼び始める
- QSO 失敗（リトライ上限）→ 自動で Watch に戻る
- 手動で Watch に戻って DF を変更することも可能

**Watch / Call の切り替え:**

Snipe ビュー上部の `[Watch] [Call]` タブで切り替え。リストはクリアされず維持されます。

## ウォーターフォール

200-2800 Hz のリアルタイムスペクトログラム。

| 要素 | 説明 |
|------|------|
| **上部ラベル** | 周波数軸（200, 500, 1000, 1500, 2000, 2500 Hz） |
| **赤い点線（縦）** | DF（TX 周波数）— 全モードでタップ時に表示 |
| **水色の帯** | Snipe Call フェーズの 500 Hz BPF 窓 |
| **赤い点線（横）** | ピリオド境界（15 秒区切り） |
| **黄色テキスト** | デコードされたメッセージ |
| **タップ / クリ���ク** | DF 設定 + ステータスバーに周波数表示 |
| **WAV ドラッグ＆ドロップ** | オフライン解析（ライブ中でも自動停止して処理） |

## ヘッダー表示

```
rs-ft8n [Scout][Snipe]   12s (213ms)
```

- **モードタブ**: Scout / Snipe 切り替え
- **残り秒数**: 次のピリオド境界までの秒数（整数）
- **デコード時間**: 直前のデコード処理にかかった時間（ms）

## TX アクション（ボトムバー）

状態に応じてボタンが動的に変わります:

| QSO 状態 | Auto ON | Auto OFF |
|----------|---------|----------|
| **IDLE** | `[CQ]` ボタン | `[CQ]` ボタン |
| **QSO 中** | 次の TX メッセージ 1 つ（タップで即送信） | 全 TX 選択肢をボタン表示 |

- **Snipe**: ターゲットと異なる局（call2）を呼ぶ `[Call xxx]` ボタンも表示
- **Halt**: 送信を即座に停止
- CQ はタップ時にキューに入り、次のピリオド開始時に送信される

---

## QSO ステートマシン

4 段階の状態遷移で QSO を管理:

```
IDLE -> CALLING -> REPORT -> FINAL -> IDLE (完了)
```

| 状態 | 意味 | 送信内容 |
|------|------|----------|
| **IDLE** | 待機中 | -- |
| **CALLING** | 呼び出し中 | `DX MYCALL GRID` or `CQ MYCALL GRID` |
| **REPORT** | レポート交換中 | `DX MYCALL R+00` |
| **FINAL** | 確認待ち | `DX MYCALL RR73` or `73` |

- **Auto ON**: 状態遷移は全自動。応答がなければリトライ（最大 15 回）。
- **Auto OFF**: TX メッセージ選択ボタンが表示され、手動で選択。
- リトライ上限に達すると QSO は incomplete としてログに記録。

## 適応バジェット（Scout モード）

Scout モードでは、デコード時間が FT8 バジェット（2.4 秒）を超えると、重い機能から順に自動で一時停止します:

| 優先度 | 機能 | 負荷 |
|--------|------|------|
| 1（先に OFF） | Multi-pass subtract | 3 パス SIC（最も重い） |
| 2（次に OFF） | A Priori (AP) | Sniper + EQ 1 パス |

ステータスバーに `[-sub]` や `[-sub,AP]` と表示されます。

デコード時間がバジェットの 60% 以下に戻ると、逆順で復帰:
1. AP を先に復帰
2. 余裕があれば subtract を復帰

Snipe モードは帯域が狭く高速なため、常に両方有効です。

## コールサインハッシュ解決

FT8 メッセージの一部（Type 4 非標準コール、DXpedition メッセージ等）では、コールサインが 10/12/22 ビットのハッシュとして送信されます。通常は `<...>` と表示されますが、rs-ft8n は**デコード済みコールサインのハッシュテーブル**を自動で構築し、以降のメッセージで `<JA1ABC>` のように実コールを表示します。

テーブルは WASM セッション中に蓄積され（最大 1000 エントリ、LRU）、ページリロードでクリアされます。

## 設定パネル

歯車アイコンで開閉。

| 項目 | 説明 |
|------|------|
| **My Callsign** | 自局コールサイン |
| **My Grid** | グリッドロケータ（4 桁） |
| **Audio Input** | 受信用オーディオデバイス |
| **Audio Output** | TX 音声の出力先デバイス |
| **DX Call (AP)** | ターゲット局コール（AP デコード + Snipe ターゲット） |
| **CAT Protocol** | Yaesu (FTDX10) / Icom (CI-V) |
| **Connect CAT** | Web Serial でリグに接続（PTT 制御） |
| **Start / Stop Audio** | ライブデコードの開始・停止 |
| **Reset QSO** | QSO を中断（incomplete としてログに保存） |
| **Open WAV File** | WAV ファイルを選択してオフライン解析 |
| **Multi-pass subtract** | 逐次干渉除去（3 パス SIC）。デフォルト ON |
| **A Priori (AP) decode** | AP デコード。デフォルト ON。Scout ではバジェット超過時に自動停止 |

## ログ管理

### 自動保存

- **QSO 完了時**: 自動で localStorage に保存（state = IDLE）
- **QSO 中断時**: Reset ボタン or リトライ上限で incomplete として保存（state = CALLING/REPORT/FINAL）
- **全受信メッセージ**: 全デコード結果を RX ログとして蓄積（最大 10,000 件）

### ZIP エクスポート

設定パネルの **Export ZIP (ADIF + RX)** ボタンで、以下の 3 ファイルを ZIP ダウ���ロード:

| ファイル | 内容 |
|----------|------|
| `qso_complete_YYYYMMDD.adi` | 完了 QSO のみ（LoTW / HAMLOG 投入用） |
| `qso_all_YYYYMMDD.adi` | 全 QSO（incomplete には `<COMMENT>incomplete:STATE</COMMENT>` 付き） |
| `rx_YYYYMMDD.csv` | 全受信デコードログ（UTC, Freq, SNR, Message） |

### ログのクリア

**Clear All Logs** ボタンで QSO ログと RX ログの両方を削除（確認ダイアログあり）。

## WAV ファイルでのオフライン解析

ウォーターフォールに WAV ファイルをドラッグ＆ドロップするか、設定パネルの **Open WAV File** で選択。

- **要件**: 12 kHz / 16 bit / mono の WAV
- ライブデコード中でも自動停止して処理
- ウォーターフォール + デコード結果が即座に表示

---

## CAT 制御

Web Serial API を使い、ブ��ウザからリグの PTT を制御します。

**対応プロトコル:**

| プロトコル | リグ例 | ボーレート |
|-----------|--------|-----------|
| **Yaesu** | FTDX10, FT-991A 等 | 38400 |
| **Icom CI-V** | IC-7300, IC-705 等 | 19200 |

**接続手順:**

1. USB ケーブルでリグと PC を接続
2. 設定パネルで CAT Protocol を選択
3. **Connect CAT** → ブラウザのシリアルポート選択ダイアログが開く
4. ポートを選択 → 接続完了
5. TX 時に自動で PTT ON/OFF

> Web Serial API は Chrome / Edge でのみ利用可能です。Safari / Firefox は非対応。

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| 「Select audio device」と表示される | 設定パネルで Audio Input を選択してください |
| デコード結果が 0 件 | アンテナ接続、オーディオレベル、周波数（14.074 MHz 等）を確認 |
| WAV ドロップでエラー | 12 kHz / 16 bit / mono か確認。48 kHz WAV は非対応 |
| CAT 接続できない | Chrome / Edge を使用。ブラウザ再起動後に再試行 |
| ウォーターフォールが真っ黒 | Audio Input が正しいデバイスか確認。Start Audio を押し直す |
| QSO が進まない | Auto チェックが ON か確認。DX Call が正しいか確認 |
| `[-sub]` `[-sub,AP]` 表示 | デコードが重いため自動調整中。バジェット内に収まれば復帰 |
| `<...>` が解決されない | 同一セッション中にその局がデコードされていない。ページリロードでハッシュテーブルはクリア |
